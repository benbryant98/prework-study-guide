HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • HTML attributes are always defined in the beginning element, never the closing element
  • The title element defines the title of the webpage
  • The style element defines CSS styles
  • The base element defines base URL for website
  • The link element connects an external resource to the HTML doc
  • The meta element defines metadata such as character set, description, keywords, author, and viewport
  • The script element embeds client-side scripts in HTML doc
  • The id attribute identifies elements and allows them to be selected

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Selectors define the element of attributes to which the rules or declarations will apply
  • Declarations contain two important components: the CSS property to apply and the value of that property (Ex color: blue)

Git

  • The git command must proceed every statement
  • The checkout command is to move the working branch to a new branch
  • The -b flag creates and new branch
  • The add command adds modifications in the current working branch to staging area
  • The -A flag indicates we want all changes
  • The pull command receives a branch's modifications into the local environment
  • The origin keyword indicates source of pull is GitHub repository
  • The push command can be used to create a new remote branch or add new changes to existing branch
